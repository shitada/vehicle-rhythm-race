# のりものリズムレース（Vehicle Rhythm Race）仕様書

## Overview
既存「のりものリズムパレード」(https://shitada.github.io/vehicle-rhythm-parade/) と同じデザインテイストで、横スクロール型リズムレースゲームを新規作成。
プロシージャル生成メロディに同期した障害物をジャンプ回避。後半に2段ジャンプ障害物、コイン収集、ノリノリモード、乗り物アンロック、固有ゴール演出。

## Requirements
- 対象年齢: 4〜5歳
- ゲーム形式: 横スクロール型レース
- 技術: HTML/CSS/JS のみ（外部ライブラリなし）
- 乗り物: 既存10種引き継ぎ（アンロック制: 初期3種）
- デザイン: 絵文字ベース、パステルカラー、角丸UI、ひらがな
- メロディが主→障害物が従（プロシージャル生成、毎回異なるプレイ体験）
- お手本なし（障害物が視覚的に迫るのでタイミングは見てわかる）
- 2段ジャンプ障害物（後半ラウンド、⚠️注意表示付き）
- コイン⭐収集（休符拍に配置、ジャンプ動機付け）
- ノリノリモード（コンボで加速演出）
- 乗り物固有ゴール演出
- バッキングトラックあり（ドラム＋ベースのループ伴奏）

## ゲームコンセプト
プレイヤーが選んだ乗り物が画面左から右へ自動で進む。道中にランダム生成されたメロディの音符に合わせて障害物が出現し、タップでジャンプ回避する。タイミングが良いほど高得点。各乗り物ごとにテーマの異なるコース・BGM・障害物が登場し、全コースクリアを目指す。

### 教育要素
- リズム感・タイミング感覚を養う
- メロディを聴いて反応する力を育てる
- 乗り物ごとに異なるテンポで段階的に難易度UP

## メロディ生成アーキテクチャ

### 生成フロー
1. 乗り物テーマ（音階・音色・BPM）読み込み
2. 音階からランダムに音符選択（跳躍制限付き）
3. リズムパターンプールからランダムにリズム選択
4. 音符列 = [{time, note, type}, ...] のシーケンスを生成
5. 各音符の time に合わせて障害物を配置
6. ラウンド進行で密度UP・テンポUP

### 音楽的制約（ランダムだけど心地よい）
- 乗り物ごとに音階を固定（ペンタトニック/メジャー/マイナーなど）
- 音程の跳躍を制限（隣接音 or 3度以内が80%、大きな跳躍は20%）
- フレーズ構造を維持（4拍単位でフレーズを区切り、フレーズ末は長い音符で着地感）
- 休符を適度に挿入（連続タップ上限を制御、子どもが追いつける間隔を確保）
- 2段ジャンプ用: 連続する2つの八分音符ペアを生成（後半ラウンドで出現率UP）
- コイン配置: 休符の拍にコイン音符を配置（メロディとは別の高い音）

## Phases

### Phase 1: 基盤構築
1. プロジェクト構造 — index.html, style.css, game.js, audio.js, melody-generator.js, courses.js
2. デザインシステム — 既存サイト踏襲（パステルグラデーション、角丸ボタン、大きな絵文字、ひらがな）
3. 画面遷移 — タイトル → 乗り物選択 → ゲーム → リザルト のステートマシン

### Phase 2: プロシージャルメロディエンジン（depends on Phase 1）
4. 音階定義 — 乗り物ごとにスケール・基本BPM・音色パラメータをcourses.jsに定義
5. メロディシーケンス生成 — melody-generator.jsの核心アルゴリズム
6. Web Audio APIシンセサイザー — audio.jsに乗り物ごとの音色
7. バッキングトラック — ドラム＋ベースラインをBPMに同期してループ再生

### Phase 3: コアゲームエンジン（depends on Phase 2）
8. 横スクロールエンジン — CSS transform + requestAnimationFrame、パララックス
9. 障害物＆コイン配置 — 通常/2段ジャンプ/コイン
10. タップ判定 — ±250ms=Perfect, ±450ms=Good
11. ジャンプシステム — 通常＋2段ジャンプ
12. ノリノリモード — コンボで加速演出

### Phase 4: コンテンツ（parallel with Phase 3）
13. 10種の乗り物テーマ
14. アンロック順序
15. ラウンド難易度
16. スター＆コンボ

### Phase 5: UI/UX（depends on Phase 3, 4）
17-21. タイトル/選択/ゲーム/リザルト画面、レスポンシブ

### Phase 6: 演出＆ポリッシュ（depends on Phase 5）
22-26. エフェクト/ノリノリ演出/ゴール演出/チュートリアル/localStorage

## Decisions
- メロディが主、障害物が従
- 2段ジャンプ: R2後半〜、⚠️「2かい！」ヒント付き
- コイン: 休符拍に配置
- ノリノリモード: コンボ5+で加速
- アンロック: 初期3種、BPM低い順解放
- ゴール演出: 乗り物固有
- お手本なし
- 判定窓広め: ±250ms=Perfect, ±450ms=Good
- 操作: タップのみ（1回 or 2段2回）
